## Who We Are

Team Castaldi is the development home of **Nathan** — an IT professional, homelab builder, and lifelong problem-solver working at the intersection of infrastructure, identity, and community.

Every project here began the same way: something was broken, missing, or harder than it needed to be. These are the solutions.

Work here is built with intentionality — requirements and architecture are documented before a single line of code is written, security and data privacy are treated as first principles, and nothing ships until it's something worth shipping.

### 🏠 Homelab MCP Server + Discord Bot
**`homelab-registry-mcp`** · **`homelab-discord-bot`** · *Private — In Development*

A Model Context Protocol server and Discord-native interface for homelab management. Built for infrastructure operators who want to reason through complex problems — security posture, service health, configuration drift — in plain language.

Features an agentic monitoring loop that observes the lab, surfaces anomalies, and — when approved — pushes remediation changes and triggers service redeployment. Infrastructure management as a conversation, not a chore.

### 🔎 Gooshie — Job Search Automation Platform
**`gooshie`** · *Private — In Development*

A modular job search automation platform that uses LLMs to find, verify, and rank opportunities against a defined candidate profile — cutting through ATS-optimized noise to surface roles that actually fit.

Built out of firsthand frustration with the modern hiring process. If the system is going to be automated on the employer side, it should be automated on the candidate side too.

### 🛼 Atlanta Junior Roller Derby — Membership App
**`ajrd-ops`** · *Private — In Development*

A volunteer project for the Atlanta Junior Roller Derby league. Replaces a fragmented paper-and-spreadsheet membership process with a proper app: registration, waiver tracking, facility check-in, and roster management — all in one place.

Built for real people, for free, because the [league](https://atlantarollerderby.com) deserves tools that work.

### 🐾 clasp.pet — Pet Tag & Finder Platform
**`clasp.pet`** · *Private — Early Stage*

A platform built around the idea that a lost pet's tag should do more than display a phone number. Physical tags (3D-printed or manufactured) linked to digital profiles: contact information, medical notes, and a self-service finder flow that doesn't require the finder to download anything or create an account.

Early stage. The problem is simple. The solution is almost as simple.

### 📺 Local Tech Pro — Managed Entertainment Platform
**`tv-as-a-service`** · *Private — Early Stage*

A B2B2C platform turning in-room entertainment in short-term rentals into a managed subscription service. Premium hardware plus 24/7 concierge support for property owners; friction-free self-service troubleshooting for their guests.

The gap between "smart home" and "reliably working home" is a real problem in the STR space. This is the bridge.

## How This Gets Built

Every project in this org follows the same workflow:

1. **Define the problem clearly** — requirements and architecture are documented before any code is written. If it can't be explained precisely, it isn't ready to be built.
2. **Security by default** — PII handling, access controls, and data minimization are considered at design time, not added later.
3. **AI-assisted, human-directed** — Claude and other LLMs are active parts of the development workflow. This isn't something to obscure: they accelerate execution, surface alternatives, and help maintain documentation discipline. The ideas, architecture, and judgment are mine.
4. **Iterative and honest** — projects ship when they're ready. Status is communicated clearly rather than hidden behind polished marketing language.
